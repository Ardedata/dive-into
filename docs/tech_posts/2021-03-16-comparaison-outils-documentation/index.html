🚀 Objectif de ce post 🚀#

Comparer plusieurs outils en fonctions des besoins définis ci-dessous !

Il n'y aura qu'un seul gagnant 🥇 !

Nos besoins#

On veut :

  • Quelque chose de sexy, qui donne envie (et non un skyblog so 2009)
  • Une barre de recherche
  • Le déployer sur un stockage objet (s3) ou sur du gitlab/github pages
  • Que ça soit simple à mettre en place (max 1/2 journée)
  • Que ça soit juste un markdown à ajouter/modifier/supprimer

Pour ceux qui ne sont pas intéressés aux détails vous pouvez directement aller à la rubrique résultats

💻 Les candidats 💻#

Les outils que nous avons retenus sont des projets OpenSource.

Sphinx 🐱#

On le retrouve beaucoup pour les documentations python, lui même écrit en 🐍 !

(voir les templates: https://sphinx-themes.org/ , ils vous sont surement familiers 😉)

Il génère des fichiers statiques (css,js,html) à partir de sources reStructuredText.

Qu'est-ce que le format .rst ?

Le format .rst est un langage de balisage léger

Pour comprendre les différences entre le markdown et le reStructuredText, nous vous invitons à consulter cet article

Hugo#

Vendu comme : The world’s fastest framework for building websites

Hugo est un générator de site web ecrit en Go

Il génère du contenu statiques à partir du contenu qu'on a écrit en markdown 🤓 ( comme un README.md avec des metadata )

Docusaurus 🦖#

Développé par Facebook, il est utilisé pour les documentations de React, React Native, Redux...

Docusaurus est écrit en js/ts (react) et permet de générer un site avec du contenu écrit en markdown.

Des examples réalisés avec docusaurus : https://docusaurus.io/showcase

D'autres solutions que nous n'avons pas retenues#

Mkdocs

Gatsby

GitBook
