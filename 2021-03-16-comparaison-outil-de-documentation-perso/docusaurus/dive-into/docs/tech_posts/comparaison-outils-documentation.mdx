## 🚀 Objectif de ce post 🚀 

Comparer plusieurs outils en fonctions des besoins définis ci-dessous !

 Il n'y aura qu'un seul gagnant 🥇 !

Nos besoins
-----------

**On veut :**

+ Quelque chose de sexy, qui donne envie (et non un skyblog so 2009)
+ Une barre de recherche
+ Le déployer sur un stockage objet (s3) ou sur du gitlab/github pages
+ Que ça soit simple à mettre en place (max 1/2 journée)
+ Que ça soit juste un markdown à ajouter/modifier/supprimer


Pour ceux qui ne sont pas intéressé aux détails vous pouvez directement aller à la rubrique [résultats](#résultats)

## 💻 Les candidats 💻

Les outils que nous avons retenus sont des projets OpenSource.

[Sphinx](https://github.com/sphinx-doc/sphinx) 🐱
-------------------------------------------------

[![GitHub stars](https://img.shields.io/github/stars/sphinx-doc/sphinx?style=social&label=Star&maxAge=2592000)](https://github.com/sphinx-doc/sphinx/stargazers)

On le retrouve beaucoup pour les documentations python, lui même écrit en 🐍 !

(*voir les templates: https://sphinx-themes.org/ , ils vous sont surement familiers* 😉)

Il génère des fichiers statiques (css,js,html) à partir de sources `reStructuredText`.

:::tip Qu'est-ce que le format .rst ?

Le format `.rst` est un [langage de balisage léger](https://www.sphinx-doc.org/en/master/usage/restructuredtext/index.html)

Pour comprendre les différences entre le markdown et le reStructuredText, nous vous invitons à consulter cet [article](https://www.zverovich.net/2016/06/16/rst-vs-markdown.html)
:::

[Hugo](https://themes.gohugo.io/)
---------------------------------

[![GitHub stars](https://img.shields.io/github/stars/gohugoio/hugo?style=social&label=Star&maxAge=2592000)](https://github.com/gohugoio/hugo/stargazers)

Vendu comme : `The world’s fastest framework for building websites`

Hugo est un générator de site web ecrit en [Go](https://golang.org/) 

Il génère du contenu statiques à partir du contenu qu'on a écrit en markdown 🤓 ( comme un README.md avec des metadata ) 


[Docusaurus](https://github.com/facebook/docusaurus) 🦖
-------------------------------------------------------

[![GitHub stars](https://img.shields.io/github/stars/facebook/docusaurus?style=social&label=Star&maxAge=2592000)](https://github.com/gohugoio/facebook/docusaurus)

Développé par Facebook, il est utilisé pour les documentations de React, React Native, Redux... 

Docusaurus est écrit en js/ts (react) et permet de générer un site avec du contenu écrit en markdown. (Comme celui-[ci](https://les-copains.gitlab.io/dive-into/docs/tech_posts/comparaison-outils-documentation/))
